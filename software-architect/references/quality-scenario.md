# Quality Scenario Structure

To make ISO 25010 characteristics concrete, use the following components:

1. **Source of Stimulus:** The stakeholder or system that triggers the event (e.g., User, Admin).
2. **Stimulus:** The actual event (e.g., "types in data," "server fails").
3. **Environment:** The context or system state (e.g., "under high load," "normal operation").
4. **Artifact:** The specific part of the system affected.
5. **Response:** The required behavior (e.g., "system stays available").
6. **Response Measure:** A quantifiable metric (e.g., "latency < 200ms," "restart within 5s").
